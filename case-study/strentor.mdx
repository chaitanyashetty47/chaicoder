## Client Overview

* Industry: Premium Wellness / High-Performance Coaching
* Business Model: Paid subscription-based product
* Stage: Growing
* Market: International (India, US, Canada, Australia, Singapore, NZ, UAE)
* Offering: 1:1 guided programs (Fitness, Psychological Coaching, Manifestation)

Unlike generic wellness apps, Strentor delivers **direct expert guidance**, making personalization and system design critical.

---

## The Challenge

When we started, Strentor had:

* A generic landing page
* Poor messaging clarity
* Weak branding structure
* Low conversion flow
* No structured SEO architecture
* No scalable client management system

Internally, operations were fragmented:

* Trainers managed clients through **Excel**
* Communication was happening over **WhatsApp**
* Zoom scheduling was manual
* Subscription upgrades/downgrades were manually handled

This created:

* Operational bottlenecks
* Poor visibility across teams
* Revenue leakage risk
* No central subscription intelligence

If left unchanged, growth would have plateaued despite increasing ad spend (~30,000 monthly visitors pre-engagement).

Strentor didn't need just a website.

They needed infrastructure.

---

## The Solution

We rebuilt Strentor as a **full-stack product ecosystem** using **Next.js + Supabase**, combining product strategy, UX design, backend architecture, and automation.

### Phase-Based Execution (Strategic Decision)

Instead of building everything at once, we executed in 3 strategic phases:

### Phase 1 – Conversion Foundation (1 Month)

* High-converting landing page
* Clear headline / subheadline / CTA structure
* Dedicated SEO-optimized service pages
* Lead capture before Calendly booking
* Structured SEO architecture

This allowed them to:

* Run ads immediately
* Build a waitlist
* Capture abandoned leads

### Phase 2 – Core Product System (3 Months)

* Client dashboard (fitness / psychological / manifestation)
* Trainer dashboard with client management
* Razorpay subscription engine
* Zoom scheduling integration
* Role-based system (Client / Trainer / Admin)
* Backend-driven subscription state management
* Supabase row-level security policies

### Phase 3 – Scaling & Intelligence (2 Months)

* Workout tracking engine
* Personal records tracking
* Weekly/overall progress analytics
* BMI, BMR, Ideal Weight calculators
* Advanced admin panel
* Feedback-based UX improvements

Then followed by:
**4 months of maintenance & scaling improvements**

## Key Features Built

### Advanced Subscription Engine

The most technically complex component.

Strentor offers:

* 3 core categories (Fitness / Psychological / Manifestation)
* Monthly / Quarterly / Annual plans
* Simultaneous subscriptions allowed
* Upgrades/downgrades
* All-in-One bundled plan

We built:

* Centralized backend subscription state manager
* Razorpay webhook synchronization
* Upgrade/downgrade handling logic
* Multi-subscription edge-case coverage
* Backend-truth architecture (frontend reads, never controls)

This prevented split-state conflicts and revenue inconsistencies.

---

### Role-Based Access Control

Using Supabase policies:

* Clients access only their plans
* Trainers see only assigned clients
* Admin has global visibility
* Secure data isolation across categories

---

### Performance & SEO Architecture

* Structured semantic SEO implementation
* Optimized page structure for crawlability
* Strategic keyword alignment
* Performance-optimized Next.js deployment
